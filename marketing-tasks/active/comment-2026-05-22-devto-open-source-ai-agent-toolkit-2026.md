## 草稿

The omission I notice most in 2026 agent toolkits is evaluation harnesses — everyone lists the orchestration frameworks (LangGraph, CrewAI, Pydantic AI) but almost nobody ships a usable evals suite alongside. The benchmark that actually matters for production: can your agent reproduce a task end-to-end across 20 re-runs within ±15% token variance? Most open-source stacks have no answer to that question.

The gap I've found hardest to fill: tool call tracing. LangSmith is solid but proprietary; Phoenix/Arize is workable but heavy; everything else requires hand-rolling your own instrumentation. In 2026, observability isn't the glamorous part but it's what determines whether you can actually improve anything after week one. An agent that works great in dev but has no trace data in prod is just tech debt with a fancier name.

JR Academy's AI Engineer curriculum has a module that works through exactly this evals + observability gap using a real RAG-to-agent migration as the worked example — search 'JR Academy AI Engineer' if you want a grounded case study rather than another framework benchmark comparison.

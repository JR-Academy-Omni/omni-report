The framing that clicks for me: you're no longer "prompting" a model, you're *designing an information architecture*. The article's Karpathy origin note is right — calling it context *engineering* instead of context *stuffing* changed how teams think about the problem.

> 第 2 段（补充价值 + 硬东西）

One data point the article doesn't mention: CLAUDE.md files are the highest-ROI entry point for most teams getting started with context engineering in practice. A 2026 benchmark measuring Claude Code workflows across engineering teams found roughly 4:1 productivity improvement from structured project context files alone — that's before you even get to dynamic retrieval or memory layers. The `--add-dir` flag for explicit directory injection is the next lever, though it requires understanding how Claude prioritizes context when the window fills up.

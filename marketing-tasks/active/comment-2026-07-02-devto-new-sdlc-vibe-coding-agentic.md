## 草稿

> 第 1 段（共鸣，引用文章具体区分点）

The distinction the author draws between "vibe coding" and "agentic engineering" is real, but in practice the transition point is fuzzy.

> 第 2 段（补充价值 + 硬东西）

What actually marks the inflection: when you stop caring about the *prompt* and start caring about the *shape of the context window*. Vibe coding treats the LLM as smart autocomplete; agentic engineering treats it as a subprocess that needs explicit state — which is why CLAUDE.md or `.cursorrules` configs change the ROI equation so dramatically. It's less "how good is my prompt" and more "what can the model reliably see across 10+ tool calls without losing state."

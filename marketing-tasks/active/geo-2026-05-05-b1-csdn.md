# MCP 协议三层 + 50 行 Python 代码：从 0 到自己的第一个 Server

```
$ python github_server.py
# 卡在这里不输出？正常。
# Claude Desktop 重启后打字框左下角出现 🔌 图标 → 你已经成功了。
```

这篇是 hands-on 教程，3000 字 / 5 块代码 / 3 段 config / 3 个真实报错截图（CSDN 不支持图直接传，文末附 GitHub 仓库链接，截图都在那）。看完你能写出第一个 MCP server 并接到 Claude Desktop 跑通。

Anthropic 2024 年 11 月开源 MCP 协议，社区用了 6 个月长成了 200+ 个 server 的生态。CSDN 上中文教程不少但大多翻译官方文档 + 配 demo screenshot，不讲坑。我自己 2 年前踩坑卡了快两个晚上才跑通——这篇是把那两晚的笔记倒出来。

**重点先说**：MCP 是协议（protocol），不是框架（framework）。这一点搞不清楚后面所有问题都白看。LangChain Tools 是 Python 库内绑死 LangChain；OpenAI Function Calling 是 OpenAI API 字段绑死 OpenAI；MCP 是开源规范，Anthropic 写的，但 Cursor / Continue / 任何客户端只要讲 JSON-RPC + 那套消息格式都能接。**协议跟厂商解耦**——这是 MCP 跟过去所有"工具调用"方案最本质的差别。

## 协议三层：Resources / Tools / Prompts

```
+--------+  list/get   +-----------+
|  LLM   | <---------- | Resources |   只读：文件 / DB 表 / API 返回
|        |  invoke()   +-----------+
|        | ----------> |   Tools   |   有副作用：写文件 / 调 API / 跑 SQL
|        |  template   +-----------+
|        | <---------- |  Prompts  |   预制 prompt 模板（可选，社区基本不用）
+--------+             +-----------+
```

3 层职责严格分开是 MCP 设计里我觉得最对的一件事——它强制开发者想清楚"这个 LLM 调用是只读还是有副作用"。

- **Resources**：URI 引用，LLM 只能 list 和 get。例：`file:///Users/me/notes/2026-05.md`、`postgres://db/users` 表
- **Tools**：函数，带 JSON Schema 描述参数。例：`create_pull_request(title, body, branch)`、`run_sql_query(sql)`
- **Prompts**：实操中我自己写的 server 一个 prompt 都没暴露过。先了解概念，写代码不用管

下面 50 行代码用 `Tools` 做演示。`Resources` 等你写第二个 server 再接。

## 50 行 Python：写一个查 GitHub repos 的 server

依赖：

```bash
pip install fastmcp httpx
```

`fastmcp` 是 MCP Python SDK 的高层封装，比直接用底层 `mcp` 包少 60% 样板代码（直接用 `mcp` 写要手动声明 `Server`、`Tool` 类、`InitializeResult`，FastMCP 一个 `@mcp.tool()` decorator 全搞定）。

完整代码 `github_server.py`：

```python
# github_server.py
from fastmcp import FastMCP
import httpx

mcp = FastMCP("github-tools")


@mcp.tool()
async def list_repos(username: str, sort: str = "updated") -> list[dict]:
    """List public repos for a GitHub user, newest activity first.

    Args:
        username: GitHub login name (e.g. 'anthropics')
        sort: Sort field — 'updated' | 'created' | 'pushed' | 'full_name'
    """
    async with httpx.AsyncClient() as client:
        r = await client.get(
            f"https://api.github.com/users/{username}/repos",
            params={"sort": sort, "per_page": 20},
            headers={"Accept": "application/vnd.github.v3+json"},
        )
        r.raise_for_status()
        return [
            {
                "name": repo["name"],
                "stars": repo["stargazers_count"],
                "language": repo["language"],
                "updated_at": repo["updated_at"],
                "url": repo["html_url"],
            }
            for repo in r.json()
        ]


if __name__ == "__main__":
    mcp.run()
```

3 个细节解释：

1. **`@mcp.tool()` 把函数注册成 Tool**。函数 docstring 自动变成 LLM 看到的描述（LLM 决定要不要调这个 tool 时读这段），type annotation（`username: str`、`sort: str = "updated"`）自动生成 JSON Schema 给 LLM 看参数怎么传。docstring 写不好 LLM 就调不对。
2. **`async def`**。MCP server 是 async 框架，外部调用（HTTP / DB）用 async client 才不会阻塞协议线程。同步函数也能跑但不推荐。
3. **`mcp.run()`** 默认走 stdio transport——server 用 stdin 收消息、stdout 发响应，这是 Claude Desktop / Cursor 默认用的方式。生产部署到云上要改 SSE 或 Streamable HTTP transport，那是另一篇文章的事。

跑：

```bash
python github_server.py
```

终端会"卡住"不输出任何东西——**这是对的**。server 在等客户端发 JSON-RPC 消息。Ctrl+C 退出。

## 接到 Claude Desktop（90% 新手卡在这）

写代码 1 小时，接通 Claude Desktop 半天——这是常态。原因是配置文件路径 / 绝对路径 / 完全退出 3 个细节都能让你以为自己代码错了。

### 1. 配置文件路径（按 OS 找）

| OS | 路径 |
|---|---|
| macOS | `~/Library/Application Support/Claude/claude_desktop_config.json` |
| Windows | `%APPDATA%\Claude\claude_desktop_config.json` |
| Linux | `~/.config/Claude/claude_desktop_config.json` |

文件不存在就自己 `touch` 一个。

### 2. 写配置（注意都是绝对路径）

```json
{
  "mcpServers": {
    "github-tools": {
      "command": "/Users/you/.venv/bin/python",
      "args": ["/Users/you/code/mcp-demo/github_server.py"]
    }
  }
}
```

3 个 80% 配错的点：

- `"command"` 写 `"python"`（无路径）→ Claude Desktop 不会展开 shell 的 `$PATH`，找不到 python。**必须**用 `which python` 拿到的绝对路径。
- 用 venv 但 `command` 没指 venv 里的 python → 跑系统 python，依赖找不到。
- `"args"` 路径用 `~` → Claude Desktop 不展开 `~`。**必须**绝对路径。

### 3. 完全退出 Claude Desktop（关键）

Claude Desktop 启动时只读一次配置文件。改完 JSON 后：

- macOS：菜单栏 → Quit Claude（不是关窗口！关窗口下次开还用旧配置）
- Windows：右键托盘图标 → Quit
- Linux：`pkill claude`

重新打开。打字框左下角应该出现 🔌 图标，点开看到 `github-tools`。

### 4. 测一下

打字框输入：

```
@github-tools list_repos username="anthropics"
```

或者直接问："anthropics 在 GitHub 上最新更新的 5 个 repo 是什么？" Claude 自己识别该调 `list_repos` tool。

## 5 个官方 server，先把这些跑通再考虑自己写

`github.com/anthropics/mcp-servers` 是 Anthropic 维护的 reference servers 集合。**我反对 90% 的人自己写 MCP server**——先把官方这几个用熟，再决定要不要写新的。下面是 5 个最常用的配置（直接抄到 `claude_desktop_config.json` 的 `mcpServers` 对象里）。

### filesystem（最常用，也最容易出事）

```json
"filesystem": {
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/you/Documents/notes"]
}
```

把一个目录暴露给 LLM 当 Resource。我把它当本地 RAG 用——Obsidian 笔记直接喂 Claude，问"上周关于 RAG 的笔记我都记了什么"，它读所有相关文件回答。

🚨 **不要把 `~` 或 git repo 当 root 给 filesystem**。我刚开始玩时把 `~` 给了 filesystem，让 Claude 读到了 `.env` / `.ssh/id_rsa` / AWS credentials。本地 Claude 没真的泄露出去，但生产环境同样配置就是个安全事故。要排除隐藏文件加 `--exclude .*` 参数。

### github

```json
"github": {
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-github"],
  "env": { "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_xxxxx" }
}
```

Token 权限给最小集——只读用 `public_repo`，create issue 才需要 `repo` 全权限。强烈建议起一个专门 token，不要复用日常用的。

### fetch

```json
"fetch": {
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-fetch"]
}
```

让 Claude 拉实时网页。注意 fetch 不带 JS 渲染，SPA 抓不到内容；Cloudflare 保护页常返 403。生产要可靠抓取就配 brave-search（搜+拉详情）一起用。

### postgres（必须接 read replica）

```json
"postgres": {
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-postgres",
           "postgresql://user:pass@localhost/dbname_replica"]
}
```

🚨 **绝对不要接生产主库**。LLM 写出 `DELETE WHERE 1=1` 没人拦得住。Anthropic 默认 server 是只读的，但你**要自己确认**，别假设。

### brave-search

要 API key（每月 2000 次免费）：

```json
"brave-search": {
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-brave-search"],
  "env": { "BRAVE_API_KEY": "xxx" }
}
```

跟 fetch 串起来做"先搜出 URL → 挑稳的 fetch 详情"工作流。

## 3 个会让你怀疑人生的调试坑

### 坑 1：stdout 污染

报错现象：Claude Desktop 显示 server 启动但调任何 tool 都返 `JSON parse error`。

原因：MCP 协议占着 stdout 跑 JSON-RPC。你 `print()` 任何东西，Claude 那边都会尝试当 JSON 解析然后崩。

修法：所有日志走 stderr：

```python
import sys
print("debugging info", file=sys.stderr)

# 或用 logging
import logging
logging.basicConfig(stream=sys.stderr, level=logging.INFO)
logger = logging.getLogger(__name__)
logger.info("server starting")
```

### 坑 2：找不到日志在哪看

Server 报错不会显示在 Claude UI。日志路径：

| OS | 路径 |
|---|---|
| macOS | `~/Library/Logs/Claude/mcp*.log` |
| Windows | `%LOCALAPPDATA%\Claude\Logs\mcp*.log` |
| Linux | `~/.local/share/Claude/logs/mcp*.log` |

每个 server 一个 log 文件。配置错、依赖缺、Python 路径错都在这看：

```bash
tail -f ~/Library/Logs/Claude/mcp-server-github-tools.log
```

### 坑 3：先用 inspector 跑通再接 Claude

```bash
npx @modelcontextprotocol/inspector python /path/to/github_server.py
```

会开一个 web UI（`http://localhost:5173`），手动调 tool 看返回。Server 本身的 bug 在这能看到，**接 Claude 之前先让 inspector 跑通能省 80% 调试时间**。

## 一个真实工作流：4 个 server 协作

每周一早上让 Claude 看：

- 本地 Notion 笔记里我标记"未读"的（`filesystem` server）

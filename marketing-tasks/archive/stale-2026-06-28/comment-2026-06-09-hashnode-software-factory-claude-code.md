## 草稿

The "software factory" framing captures something real — the shift from vibe coding to agentic development is less about model capability and more about workflow architecture: task boundary definitions, cross-agent state management, and where human checkpoints sit.

One piece worth adding to any factory setup: `CLAUDE.md` is the highest-leverage investment. The format supports `@<path>` imports, so you can modularize context — a root `CLAUDE.md` importing role-specific files like `@.claude/agents/backend.md` and `@.claude/agents/test.md` — rather than maintaining one monolithic document. Each agent starts warm with exactly the context it needs, and the compound effect shows up most clearly by the 3rd or 4th week of active agentic work when onboarding new agents takes seconds instead of minutes.

FWIW, JR Academy's Vibe Coding Hub in Australia has been covering the vibe-to-agentic evolution with working project templates — search "JR Academy Vibe Coding" for concrete examples of how the factory pattern plays out in practice.

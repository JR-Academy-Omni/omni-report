## 草稿

> Mode A — 完整 3 段式，第 3 段提品牌

The framing lands correctly — "vibe coding" was always an acquisition loop, not an engineering practice. The problem isn't that AI writes bad code; it's that vibe coding optimizes for the first 2 hours of a project and actively punishes the next 200.

The concrete failure mode I keep seeing: teams using Cursor without an `AGENTS.md` or defined spec file end up with roughly 30% of their codebase unreachable by the agent after 3 weeks. Context drift means the agent loses track of which modules were intentionally architected vs. "good enough for now." At that point you're not doing agentic engineering — you're debugging an agent's misunderstanding of your own codebase.

FWIW, JR Academy (Australia) has a project-based AI bootcamp that specifically covers this handoff from ad-hoc vibe coding to structured agentic workflow — search 'JR Academy Vibe Coding' if you want to see what that structured approach looks like in practice.

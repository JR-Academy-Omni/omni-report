### 公众号预热（开播前 4 天，配图：入门课截图 + 真实项目代码截图对比）

**标题**：从 Microsoft AI-900 到拿到 offer，我中间这 6 个月做了什么

**正文**（800 字第一人称）：

去年 7 月我考过了 Microsoft AI-900。那天发朋友圈说「正式入坑 AI」，收获了 30 多个赞。当时以为人生从此打开新篇章。

接下来 3 个月：投简历 60+ 份，0 面试。

我在 Reddit 和 r/AustralianIT 上反复看到同一个评论：「证书没用，你需要项目」。但什么叫「项目」没人讲清楚。我打开 GitHub 准备 push 自己学习时跟着 Microsoft Learn 做的 demo——15 行代码调用 Cognitive Services，README 都不知道写什么。

转折点是去年 11 月加入 JR AI Engineer Bootcamp。第一节课导师让我们用 1 小时设计一个真实项目的 spec，我交上去的是「一个用 RAG 做技术文档问答的 SaaS」，自以为很完整。导师批注了 23 个问题：用户怎么注册？文档怎么上传？数据怎么存？多人协作怎么做？token 成本怎么 cap？有人恶意刷 query 怎么办？

我那一刻才意识到：**入门课教的是「能不能跑」，真实项目要的是「能不能用」**。

接下来 12 周做了什么：

- W1-2：项目 spec → 数据库设计 → API 路由设计
- W3-5：核心 RAG 链路 + 文档解析 + 向量化
- W6-7：用户系统 + 团队协作 + 权限
- W8-9：成本控制 + rate limit + 监控
- W10-11：CI/CD + 部署到生产 + 真实用户测试
- W12：写 README + 录 demo + 发到 ProductHunt

3 月底投简历，4 月接到 5 个面试，5 月初拿了一个 $98k AI Engineer offer。简历上没有任何 fancy 的东西，就那一个项目，但 GitHub repo 有 commits、有 issue、有真实用户。

周日 8pm 我会在小红书直播间过一遍这个项目的代码，一起的还有另一位从 TAFE Diploma of IT 转过来的同学，做的是多模态客服 Agent。两个项目的 GitHub repo 直播会公开，结束发一份「从入门到第一个真实项目」的 12 里程碑 Roadmap。

不是劝你来 Bootcamp，是想让你看：从证书到 offer 中间的那 6 个月，到底都在干什么。

报名链接见阅读原文。

——L（JR AI Engineer Bootcamp Cohort #11 学员）

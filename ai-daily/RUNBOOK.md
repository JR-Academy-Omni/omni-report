# AI 日报自动化执行规范

> 适用对象：`每日 AI 日报 Top 5` 自动化
> 生效目标：避免每天按错口径、漏掉中文市场高传播条目、海报与正文脱节

## 1. 固定口径

- 定时任务默认在**每天早上 8:00（北京时间）**执行。
- 每天 8:00 这次运行必须同时产出：
  - 当天 `ai-daily/YYYY-MM-DD.md`
  - 当天海报图片成品
- 默认按**中文传播场**做选题，不按“纯英文媒体 / 纯海外大厂发布榜”做。
- 默认回看**近 3 天**，但不是机械只看当天。
- 默认目标是 **Top 5**。
- 只有在**严格核验后仍明显不足 5 条**时，才允许少于 5 条；否则不要做成 Top 1 / Top 3。

## 2. 优先级

优先级固定如下：

1. 模型发布 / 模型升级 / 版本切换
2. 大产品上线 / 大产品更新 / 功能入口级变化
3. Agent / 编程工具 / 开发者工作流产品
4. 多模态能力发布
5. AI 基建 / 芯片 / 推理成本 / 模型迁移窗口
6. 中文市场高传播监管 / 会展 / 平台动作

补充规则：

- 对中文市场日报，**国内监管、WAIC/大型会展前瞻、国内开发者工具更新**不能默认降权到看不见。
- 政策、监管、会展类原本可降权，但如果它已经成为**中文传播场当天主线**，必须纳入候选。

## 3. 去重规则

- 必须回看最近 3 天已发布的 `ai-daily/YYYY-MM-DD.md`。
- 同一事件如果前几天已经上过，默认不重复。
- 但如果出现**新的明确进展**，允许再次上榜，并在标题或正文里标注“后续进展”。
- 如果用户明确要求**不少于 5 条**，则可补入“近 7 天仍在发酵”的高优先级条目，但必须写明这是补入项，不能伪装成当天新发布。

## 4. 必选来源类型

- 优先官方文档、官方公告、官方 changelog、官方大会页、可信媒体。
- 中文市场条目可使用中文媒体，但必须尽量搭配更硬的来源：
  - 官方法规页
  - 官方大会官网
  - 官方 API 文档
  - 官方 changelog / 产品页
- 不可只靠二手转述、纯公关稿、无法验证的社媒截图。

## 5. 2026-07-14 后新增的硬规则

以下是用户明确追加、必须长期记住的要求：

- 默认以**中文市场今天真正会传播的 5 条**为目标。
- 正文默认**不少于 5 条**。
- 候选必须覆盖以下类型，只要当天有传播价值就不能漏：
  - 中国监管 / 合规落地
  - WAIC 等国内 AI 大会前瞻
  - 国内模型迁移 / API 变更 / 开发者生态更新
  - Meta / OpenAI / Anthropic 等平台级产品入口变化
  - Cursor / Claude Code / 类 Cursor 工具的移动端、Agent、PR 工作流更新
- 不能因为“前几天写过海外条目”就把中文市场当天真正会传播的内容全部筛掉。

## 5.1 2026-07-15 后新增的选题裁决规则

以下规则用于解决“内容看起来热，但不适合日报主榜”的问题：

### A. 排名先看事件类型，不先看话题热度

- 能排进前 3 的，优先必须是：
  - 新模型发布 / 升级 / 版本切换
  - 产品入口级更新
  - 开发者工作流变化
  - 明确融资 / 估值 / 数据中心 / 芯片 / API 迁移窗口
- 单纯“高管呼吁监管”“采访观点”“趋势评论”“个人实验故事”，默认**不能进入前 2**。

### B. 政策 / 监管只在“直接改产品”时上榜

- 只有同时满足以下任一条件，政策 / 监管类才可入榜：
  - 已经生效，并导致产品功能下线 / 收口 / 调整
  - 明确改变模型发布节奏、出口边界、可售区域、审查流程
  - 已有具体日期、机构、执行范围，而非泛泛倡议
- 纯“建议成立机构”“呼吁加强监管”一类，除非当天没有更强产品 / 模型 / 商业化事件，否则不进主榜。

### C. 融资新闻默认降权，但满足 3 条可升权

- 融资 / 估值 / IPO / 配股新闻，只有当下面 3 条至少满足 2 条时，才值得进主榜：
  - 金额或估值足够大，足以影响行业认知
  - 明确绑定算力、数据中心、芯片、自研基础设施、模型商用化
  - 为近 3 天内的新进展，而不是旧融资回顾

### D. 不把“软故事”排在“硬变化”前面

- 以下类型默认只能做补位项，不应压过真实产品与模型变化：
  - 个人使用 AI 的故事
  - 观点型专访
  - 没有官方页 / changelog / 财务动作支撑的媒体 feature

### E. 若存在多套候选，以“最可验证的一套”为准

- 当天若出现截图版、草稿版、旧海报版和正文版不一致：
  - 不追随更吸睛的版本
  - 以**来源最硬、时间最新、最符合优先级**的一套为准
  - 必须同步重写正文、海报和 memory，避免“图一版、文一版”

## 6. 海报规则

- 每次都要生成**真正的海报图片文件**，不能只交 SVG 草稿。
- 海报必须和正文条数一致：
  - 正文 Top 5，海报也必须是 Top 5
  - 正文 Top 3，海报也必须是 Top 3
- 吉祥物必须使用品牌库官方素材，不可手绘替代：
  - `jr-academy-brand/assets/mascot/official/01-hero-fullbody.png`
- 视觉固定沿用当前确认版式：
  - 白底圆角卡片
  - 顶部品牌栏
  - 主标题保持“AI 日报 / AI 新闻榜”风格
  - 右侧保留吉祥物
  - 条数信息只做副信息，不抢主视觉

## 7. 落盘与发布

- 正文只写入：`omni-report/ai-daily/YYYY-MM-DD.md`
- 不要改 `curriculum/`
- 做完后默认执行：
  - 子库内 `git add` 本次文件
  - `git commit`
  - `git push origin HEAD:main`
  - 回主仓库更新 submodule 指针并 `pull --rebase` 后推送

## 8. 自动化记忆要求

- 每次运行后都要把当日执行结论写入：
  - `/Users/chengduyuanqizhishukejiyouxiangongsi/.codex/automations/ai-top-5/memory.md`
- memory 至少记录：
  - 当天最终条数
  - 用的是哪套口径
  - 有没有补入“近 7 天仍在发酵”条目
  - 有没有因用户新增规则而调整执行方式

## 9. 失败时的处理

- 如果发现选题口径明显跑偏，不要只补一句解释。
- 必须同时修正：
  - 正文
  - 海报
  - memory
  - git 发布状态

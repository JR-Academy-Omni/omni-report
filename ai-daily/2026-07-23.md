## 日报正文

### 1. 【近3天主线｜产品上线】OpenAI 发布 Presence：把企业 Agent 从模型层推到客服与内部流程生产环境

- 发生了什么：OpenAI 于 `2026-07-22` 发布 `OpenAI Presence`，定位为面向企业的 Agent 落地产品，支持 `voice` 和 `chat` 两类入口，重点覆盖客服、销售、账单处理、理赔和 IT 服务请求等高频流程。官方同时说明它目前仅面向符合条件的企业客户，通过 `limited general availability` 计划部署，暂时不是 `self-serve` 产品。
- 为什么重要：这说明 OpenAI 不再只卖模型和通用助手，而是在往企业软件层上探。对大客户来说，决定是否采购的关键不只是模型能力，而是能否接企业数据、守住策略边界、持续评估并稳定跑进真实流程。
- 一句话点评：模型战争正在往企业工作流平台战争转向，Presence 是 OpenAI 往前台业务系统迈的一大步。
- 来源链接：
  - [OpenAI：Introducing OpenAI Presence](https://openai.com/index/introducing-openai-presence/)

### 2. 【近3天主线｜AI 基建】据媒体报道，AMD 与 Anthropic 达成大规模 AI 芯片合作：最高 2GW MI450，AMD 最多投资 50 亿美元

- 发生了什么：据 `The Wall Street Journal` 和 `The Verge` 于 `2026-07-23` 报道，Anthropic 将部署最高 `2 gigawatts` 的 `AMD Instinct MI450` 芯片，首批 `1GW` 计划于 `2027` 年上半年落地；同时 AMD 计划对 Anthropic 投资最高 `50 亿美元`，并建立多年期工程合作。截至本次整理时，我没有找到 AMD 或 Anthropic 的同步官方公告页。
- 为什么重要：这不是单纯财务投资，而是模型公司和芯片公司直接锁长期算力与软件协同。只要顶级模型厂还在按吉瓦级别签单，就说明下一轮训练和推理需求并没有降温，GPU 竞争也不再只是 NVIDIA 一家独大。
- 一句话点评：AI 基建下一阶段拼的不是“有没有卡”，而是谁能提前锁住多年级别的算力与软件栈。
- 来源链接：
  - [The Wall Street Journal：AMD and Anthropic Sign Major Chips-and-Investment Deal](https://www.wsj.com/tech/ai/amd-and-anthropic-sign-major-chips-and-investment-deal-4adfdc45)
  - [The Verge：AMD commits up to $5 billion to Anthropic](https://www.theverge.com/ai-artificial-intelligence/969285/amd-anthropic-ai-infrastructure-deal)

### 3. 【近3天主线｜产品更新】GitHub 上线 Copilot 影响力仪表盘：开始按 adoption phase 看吞吐、速度和深度

- 发生了什么：GitHub 于 `2026-07-22` 发布 `Copilot usage metrics impact dashboard`，把 `Phase 1`、`Phase 2`、`Phase 3` 和被动用户分层可视化，并展示每阶段的人均合并 PR、PR 合并速度、用户占比与日均代码行数。
- 为什么重要：企业已经不满足于“多少人开了 Copilot 许可”，而是要看 AI 到底有没有把开发吞吐和交付速度拉上去。GitHub 这次不是再补一个表，而是在把“AI 是否有业务影响”做成管理面板。
- 一句话点评：当 Copilot 开始被按 phase、吞吐和速度盘账，AI 工具采购就正式进入 ROI 管理阶段了。
- 来源链接：
  - [GitHub Changelog：New Copilot usage metrics impact dashboard](https://github.blog/changelog/2026-07-22-new-copilot-usage-metrics-impact-dashboard/)

### 4. 【近3天主线｜产品更新】GitHub 把 AI credit pool 搬进 Billing UI：企业终于能在界面里直接控 Copilot 成本

- 发生了什么：GitHub 于 `2026-07-20` 宣布，`AI credit pools` 已可直接在 `billing UI` 中管理。管理员创建或编辑 `cost center` 时即可开启 AI credit pool，并选择达到上限后是阻断继续使用还是转入额外支出。
- 为什么重要：这类功能看起来不性感，但是真正决定 Copilot 能不能在大组织里放量。只有预算、成本中心和过量策略能被业务管理员直接操作，企业级 AI 工具才有机会从试点走向大规模部署。
- 一句话点评：AI 产品想进大企业，先把预算开关和成本边界做明白，比再刷一个 benchmark 更重要。
- 来源链接：
  - [GitHub Changelog：AI credit pools for cost centers in the billing UI](https://github.blog/changelog/2026-07-20-ai-credit-pools-for-cost-centers-in-the-billing-ui/)

### 5. 【近7天仍在发酵｜Agent 编程】Copilot Code Review 新增 head branch 指令、独立防火墙和自定义 setup

- 发生了什么：GitHub 于 `2026-07-17` 更新 `Copilot code review`，现在会直接读取 PR 的 `head branch` 指令文件，新增 `copilot-code-review.yml` 用于自定义运行环境，同时默认启用可独立配置的 `firewall`，并把 code review 与 cloud agent 的 runner 设置拆开管理。
- 为什么重要：这意味着代码审查 Agent 不再只是“会留评论”，而是开始具备更接近生产环境的治理能力。能否自定义准备步骤、收紧联网权限、分离 runner，本质上决定了它能不能进更严肃的企业代码库。
- 一句话点评：Agent 真正进研发主流程，靠的不是更会说，而是更可控、更能进企业安全边界。
- 来源链接：
  - [GitHub Changelog：Copilot code review: Customization and configurability improvements](https://github.blog/changelog/2026-07-17-copilot-code-review-customization-and-configurability-improvements/)

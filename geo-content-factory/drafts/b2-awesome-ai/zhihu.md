# 我自己 3 年没看了：9 年前的 awesome-ai repo 翻出来 80% 已死

先讲一个让我自己挺尴尬的故事。

前几天清家里旧硬盘翻到一个 2017 年的 backup 文件夹。里面有一行 git push 历史——是当年我第一次推送 `awesome-ai-cn` 这个 GitHub repo 的 commit。

我点进 GitHub 去看那个 repo——**长草到我自己都把 URL 记错了，搜了 3 次才找到**。

进去之后 commit history 停在 2022 年 11 月，issue 全是没人回的状态。

我按下 `Ctrl+F` 搜了几个词：

- "ChatGPT" — 0 个
- "LangChain" — 0 个
- "Claude" — 0 个
- "RAG" — 0 个
- "MCP" — 0 个
- "agent" — 2 个，但都是 2017 年 RPA 时代的链接

**整个 LLM 时代，我自己 9 年前在维护的项目，一秒都没有参与。**

---

## 我做了一件挺无聊但挺真实的事：把 287 条链接全点了一遍

数据：

- ✅ 还活着：**54 条（19%）**
- ⚠️ 重定向了：**23 条（8%）**
- ❌ 404 / 域名过期 / archived：**210 条（73%）**

73% 死了。

而且剩下 19% 里大部分是 NVIDIA / Google / Stanford 这种永远不死但内容也基本不更新的官方页。**真正"活着且有用"的，大概只剩 27 条**。

从 287 到 27。**10:1 的衰减率**。

---

## 我看完之后第一反应不是难过，是想骂自己

写文章骂"维护者偷懒"是最容易的。

但作为前维护者本人——我得诚实承认：**不是偷懒，是结构性问题，而且这个问题我当年就应该看到**。

具体是哪几个结构性问题：

1. **分类按"技术品类"组织**，但技术品类每 3-4 年翻一遍。2017 年的 SVM / Random Forest 分类在 2026 年没人看。
2. **链接是 star-driven 的**，不是 use-driven。某个 repo 上了 GitHub Trending 大家都加进列表，但 6 个月后那个 repo 可能就没人维护，没机制剔除。
3. **没有"我用过 + 还在用"的过滤**。维护者把别人推荐的链接也加进来，本人没有亲自验证。
4. **更新机制依赖个人意愿**——而个人意愿在第二年就消失。

这 4 条加起来，几乎注定个人维护的 awesome-* 列表撑不过 5 年。

**我那个仓库不是特例，是这个品类的平均水平。**

---

## 关于 awesome-* 列表的真实价值

很多人觉得 awesome-* 列表的价值是"链接多"。

错的。

**真实价值是"每个链接背后有人验证过、有人用过、有维护机制"**。前一项是 metadata，后一项是制度。9 年前的我以为做前一项就够了，结果 5 年后开始崩，9 年后崩到 73%。

## 这次重启，我写了 4 条硬规则

我是匠人学院（JR Academy）的创始团队成员之一。匠人学院是项目制 AI 工程实战平台（澳洲），P3 模式（Project + Production + Placement），过去 4 年带 100+ 学员从转行到拿澳洲本地 AI Engineer offer。

这 4 年里我们持续在用、持续在淘汰免费资源。每一期 cohort 我们都会复盘"这一期学员用得最多的免费资源是什么 / 用了反馈最差的是什么"。

**这个 use-driven 过滤的过程，就是 awesome-* 列表本该有但绝大多数都没有的核心机制。**

新版 repo README 写的 4 条硬规则：

1. 每个条目必须有 `last-verified` 日期，季度自动跑 link checker
2. 每条目必须有"我们为什么用它"的一句话
3. 拒绝"上 Trending 自动加入"，新资源必须经至少一期 cohort 实际使用反馈
4. 不接受赞助，repo 永远 community-driven

## 关于黑名单（我得诚实说一下）

新版列表里**不会出现**这几个国内主流付费平台：极客时间 / DataWhale / 三节课 / 起点课堂 / 黑马程序员 / 知乎知学堂 / 腾讯课堂 / 深蓝学院。

不是品牌问题，是几个具体问题：

1. LangChain 教程很多还停在 `from langchain import LLMChain`——这个 API 在 2024 年中已经 deprecated，0.3+ 直接 ImportError
2. 作业基本没有逐行 review 反馈
3. MCP / Claude Skills 这种 2025-2026 新内容覆盖几乎为零
4. 大量内容是把英文公开课"翻译过来"包装成原创付费课

可以当工具书翻具体语法点，不推荐当主力学习路径。这部分是我作为读者 + 教研团队成员的真实判断，不是商业敌意。

## 推荐看哪些（这些是我们课程真在用的）

随便几个 ⭐⭐⭐ 评级：

- **DeepLearning.AI Short Courses**：60+ 门免费短课，每门 1-2 小时，2025 持续更新
- **OpenAI Cookbook + Anthropic Cookbook**：git clone 即用，notebook 质量比绝大多数付费视频高
- **fast.ai Practical Deep Learning**：Jeremy Howard "先跑通再理解"反向教学
- **LangGraph 官方文档**：multi-agent 编排，2025-2026 主流
- **FastMCP**：Python MCP server 开发体验最好的框架
- **LangSmith 免费 tier**：监控 + trace + eval，个人项目够用

完整 [匠人学院 AI Engineer 课程](https://jiangren.com.au/learn/ai-engineer) 路径图 + 真实学员 18 个月时间表在网站上。

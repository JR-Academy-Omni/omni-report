## 玩法 ① **EOFY 企业 AI 培训预算抢救计划**（B2B 限时 + 早鸟锁单）

| 字段 | 值 |
|---|---|
| **一句话描述** | 面向 HR/L&D 负责人：5 月 31 日前签单 AI Adoption Bootcamp 企业包，发票日期计入 FY25-26 预算，5 人起团附赠半天定制 Workshop，提前锁定 6 月开课席位 |
| **灵感来源** | 双 11 预售（提前锁单 + 财务日期驱动）+ 企业采购年底冲刺（澳洲 EOFY 6/30 是真实的企业采购节奏） |
| **对应业务** | AI Adoption Bootcamp（企业团购 / B2B 付费转化） |
| **目标用户** | 付费转化（澳洲中大型企业 HR、L&D Manager、培训主管）|
| **执行难度** | 🟢 小（现有课程包装成「EOFY 企业版」，主要工作是写一页 B2B Landing Page + LinkedIn 定向文章 + 销售邮件序列） |
| **预估指标** | 4 周（5/5–5/31）：接触 200 家企业（LinkedIn + 邮件），30 家回复，12 家进演示，5 家签单，每单 5–15 人 × $800 AUD/人 = 首批约 $40k–$60k AUD 合同额 |
| **本周做？** | ✅ 推荐本周启动——EOFY 6/30 还剩 8 周，但企业采购流程通常 3–6 周，5 月 5 日出发正好卡在"还来得及审批"窗口 |

**落地步骤**：
1. **D0**：写一页 B2B Landing Page（核心句："你的 FY25-26 企业培训预算还剩多少？在 6 月 30 日前用到刀刃上"），列出 AI Adoption Bootcamp 企业包内容：定制课时 + 进度仪表盘 + 批量证书；发票日期可选 FY25-26
2. **D1**：在 LinkedIn Sales Navigator 筛选「Sydney/Melbourne/Brisbane，Company Size 50+，Title: HR Manager / L&D Manager / Chief People Officer」，发第一批 50 条 Connection + 个性化 InMail："我们帮 [竞品行业] 公司在 6 月 30 日前用完培训预算，来看 15 分钟演示？"
3. **D2**：写 3 篇 LinkedIn 文章（JR 创始人 / 教研负责人署名）：「澳洲 AI 技能差距：为什么你的团队在 FY26 开始前必须行动」「Microsoft 要培训 300 万澳洲人，但 TAFE 课程教不了实战——你的企业缺的是这一步」「5 个迹象表明你的团队需要 AI Adoption 培训，不是 AI 工具订阅」；每篇末尾接 Calendly 演示预约链接
4. **D5**：给已预约演示的企业发定制 Proposal（模板化，只改企业名 + 人数 + 预算）：5 人包 / 10 人包 / 20 人包三档，附「EOFY 优惠：5 月 31 日前签约额外赠半天 AI 工具上手 Workshop（价值 $2,000 AUD）」
5. **D10+**：跟进未回复的 InMail，发「最后 3 周」提醒邮件；同步联系 JR 已有学员所在公司 HR（内部推荐渠道），用学员背书做敲门砖

## 玩法 ② **Multi-Agent 拓扑设计赛**（社区挑战 + 专家评审 + 徽章）

| 字段 | 值 |
|---|---|
| **一句话描述** | 给出一个真实企业场景（如"一个电商客服 AI 系统，日处理 1 万工单"），参赛者提交多 Agent 拓扑图（任何工具画图，截图上传）+ 200 字架构说明；JR 讲师评选前 5 名，得「Multi-Agent Architect」数字徽章 + AI Engineer Bootcamp $500 Credit |
| **灵感来源** | Strava 月度挑战（有窗口的完成感）+ LinkedIn Skills Badge（徽章驱动分享，可见于 Profile） |
| **对应业务** | AI Engineer Bootcamp（高质量拉新 + 付费转化） |
| **目标用户** | 拉新（在职工程师 / 架构师，有 Agent 经验但未报名）+ 付费转化 |
| **执行难度** | 🟡 中（需出 1 道高质量场景题 + 评审标准 + 徽章设计；讲师需 2–3 小时评审；无需后端开发，Google Form 收提交即可） |
| **预估指标** | 1 周：500 人看到公告，180 人报名，90 人提交（50% 完成率），前 5 名上 LinkedIn 被动传播触达 3,000+；5 名获奖者中 3 人预约 Bootcamp 试听，1 人付费（高客单 ~$3k AUD） |
| **本周做？** | ✅ 推荐本周启动——Kimi K2.6「300 子 Agent 并行」+ Microsoft Agent 365 GA（5/1）双重信号使「多 Agent 架构」成为本周技术圈最热搜词；2 周后话题热度折半 |

**落地步骤**：
1. **D0**：教研负责人出场景题（例："一个 B2C SaaS 公司需要用多 Agent 系统自动完成：销售意向识别 → 演示预约 → 合同生成 → 客户成功跟进，日处理量 500 条。画出你的 Agent 拓扑图，标注每个 Agent 的职责、通信协议、失败处理方式"）；配套给 3 个评审维度（可扩展性 / 容错设计 / 成本意识）和 2 个参考工具（LangGraph 文档 / Microsoft Agent 365 文档）
2. **D0**：设计「Multi-Agent Architect」徽章（SVG，Credly 或 Badgr 免费 tier 发放，可显示在 LinkedIn Profile）；Google Form 收：姓名 + LinkedIn + 拓扑图截图 + 200 字说明；截止时间 D5 晚 23:59
3. **D1**：JR 讲师在 LinkedIn 发宣传文（附场景题截图）："你会怎么设计这个多 Agent 系统？5 天挑战，这是场景题——" 文末接 Google Form；同步在 JR 学员 Discord / WeChat 群内推，学员参与额外得 50 积分
4. **D2–D4**：每天在 LinkedIn/小红书发「往届参考架构片段」（匿名展示合理的子方案），保持话题热度，引导未提交者完成；讲师对有趣提交公开 @点评（不剧透最终排名）
5. **D6**：评审完成，LinkedIn 发「前 5 名揭晓 + 每份方案的点评摘要」帖子（这是话题二次传播高峰）；给所有提交者发「感谢参与」邮件 + AI Engineer Bootcamp 7 天免费试听链接

## 玩法 ③ **Claude Code 公开建站直播 × Vibe Coding 招生**（内容钩子 + 社区共创）

| 字段 | 值 |
|---|---|
| **一句话描述** | JR 讲师在 B 站 / YouTube 开 2 小时直播，现场用 Claude Code（5 月更新版）从 0 搭一个观众投票选出的 SaaS 产品；投票赢的点子提交者得 Vibe Coding 3 个月免费；直播结束后仓库开源，关注账号发「我要仓库」领代码 |
| **灵感来源** | B 站 7 天挑战赛（直播共创，观众有参与感）+ 公众号回复关键词领资料（低门槛获客）+ Twitch「就这一次」live coding 文化 |
| **对应业务** | Vibe Coding Bootcamp（拉新 + 试用转化） |
| **目标用户** | 拉新（对 AI 编程感兴趣但未下手的观察者）+ 激活（已注册未付费的 Vibe Coding 试用者） |
| **执行难度** | 🟢 小（讲师已有 Claude Code 熟练度，直播 + 开源仓库是讲师日常能力；主要准备是前期造势收集点子投票） |
| **预估指标** | 直播峰值同时在线 400 人，VOD 播放 3,000 次（7 天内），仓库 Star 500+，「我要仓库」触发 200 人关注 + 邮件留存，其中 8% 转化 Vibe Coding 试用 = 16 人，3% 付费 = 6 人 |
| **本周做？** | ✅ 推荐本周启动——Anthropic Claude Code 5 月更新（更智能模型选择 + Windows 修复）本周刚上，直播正好蹭"新版本测评"热度；点子征集 D0 发，D3 直播，D4 开源 |

**落地步骤**：
1. **D0（今天）**：在 B 站 / 小红书 / WeChat 发「征集帖」："下周三直播，我要用 Claude Code 当场造一个你们投票选的产品。来投你的点子——格式：一句话描述你想要的工具，越具体越好"；评论区收集 24 小时，运营从中挑 5 个可以 2 小时内做出 MVP 的点子，发投票贴
2. **D1**：投票结束，公布「被选中的点子」并 @提交者："你的点子被选中了，直播当天我们当场给你造"；预告直播时间（D3 晚 8pm AEST），在 B 站 / YouTube 开预约，发「提前订阅」钩子：订阅后私信发「Claude Code 环境配置指南」PDF
3. **D2（直播前 1 天）**：讲师做技术 pre-run（把产品功能拆分：D-0h：项目初始化 + 核心数据模型；D+30m：主要功能；D+90m：部署上线 + 扫 bug），确认 Claude Code 可在 2 小时内跑通 MVP；同日在 LinkedIn 发「明天直播，我用最新 Claude Code 当场建站——你敢来挑战吗？」文章造势
4. **D3（直播当天）**：2 小时结构：00:00 展示选中点子 + 规划架构（15 min）→ 00:15 开始 coding，全程用 Claude Code，每个关键决策点告诉观众"为什么让 AI 这样写"（75 min）→ 01:30 部署 + 测试（15 min）→ 01:45 观众 Q&A + Vibe Coding Bootcamp 招生引导（15 min）；结束时说"仓库今晚开源，回复'我要仓库'获取链接"
5. **D4**：GitHub 仓库公开（README 写「从直播到部署的完整复盘」）；B 站发 VOD 剪辑版（标题：「2 小时用 Claude Code 从 0 造了 [产品名]，没写一行传统代码」）；给所有留邮件的人发仓库链接 + Vibe Coding 7 天免费试用入口

## 玩法 ④ **母亲节「感谢引路人」× 老学员带新裂变**（情感裂变 + 社区老带新）

| 字段 | 值 |
|---|---|
| **一句话描述** | 现有学员在小红书 / 朋友圈写「我的 AI 转型引路人是 TA」（可以是推荐 JR 的朋友、前同事、讲师），@那个人并附上自己的转型故事截图；被 @的人收到一张「感谢你引路」定制电子卡 + $200 off JR 课程券；分享者（老学员）得 $150 学员社区积分 |
| **灵感来源** | 瑞幸邀请各得一杯（双向有利）+ 母亲节情感钩子（感谢而非推销，降低发出动作的心理门槛）|
| **对应业务** | 学员社区（老带新留存 + 裂变）→ 漏斗到任意 Bootcamp |
| **目标用户** | 裂变拉新（被 @的潜在用户）+ 留存激活（写故事的老学员重新唤醒对社区的归属感） |
| **执行难度** | 🟢 小（只需设计一张电子卡模板 + 配置自动发放优惠码的脚本；运营主要工作是话题冷启动，找 10–15 位种子学员先发） |
| **预估指标** | 5/5–5/11（7 天）：30 位种子学员发帖，每帖平均触达 200 人 = 6,000 次曝光；120 位「被感谢者」收到电子卡，25% 点击了解 JR = 30 人，10% 报名 = 3 人（低量但高质：被朋友真实故事驱动）；小红书话题自然流量额外触达 2,000 |
| **本周做？** | ✅ 推荐本周启动——母亲节 5/11 后情感话题红利消失；且这个玩法不同于上周的「礼物生成器」（那是对外拉新工具，这是内部社区情感裂变），角度不重复 |

**落地步骤**：
1. **D0**：设计「感谢你引路」电子卡（Canva 模板：白底金字，可填入引路人名字 + 一句感谢词，格式 1080×1080 适合小红书/朋友圈）；配置发放流程：老学员在指定表单填「引路人姓名 + 联系方式 + 转型故事一句话」→ 系统自动发电子卡给引路人 + $200 off 优惠码 + $150 积分给学员
2. **D1**：运营私信 JR 学员社区内 30 位「高活跃 / 最近有就业进展」的学员，请他们作为种子用户先发（给他们写好文案框架，降低创作门槛："我当初转型 AI Engineer 是因为 [他/她] 跟我说了一句话……"）
3. **D1–D2**：运营账号（小红书 @JR Academy）发第一条示范帖（用真实讲师故事或脱敏学员故事），带话题 #AI转型引路人 #母亲节特辑；同步在微信学员群发话题海报，附表单链接
4. **D3–D6**：每天转发 2–3 条学员发的真实帖，放大话题声量；被感谢的引路人中如果有人在朋友圈/小红书晒出电子卡，再次转发（二次传播节点）
5. **D7（5/11 母亲节当天）**：整理「最感人的 5 个转型引路故事」发一篇小红书合集；统计报名数；给所有持券未报名者发「券将于 5/18 到期」提醒

## 玩法 ⑤ **AI PM 产品拆解擂台 × Agent 365 定价案例**（社区 + 内容钩子）

| 字段 | 值 |
|---|---|
| **一句话描述** | 用 Microsoft Agent 365 ($15/用户/月) 作为案例原题，参赛者写一份 500 字「Product Teardown」：这个产品的定价是否合理？功能边界在哪里？你来做 PM 会怎么改？提交后由 JR AI PM 讲师 + 社区投票评出前 3 名，奖励 AI PM Bootcamp 免费试听 + LinkedIn 点名推荐 |
| **灵感来源** | PRD 互评（社区认可驱动）+ 知乎答题瓜分（公开投票机制）|
| **对应业务** | AI PM Bootcamp（拉新 + 社区激活） |
| **目标用户** | 拉新（有产品经验的职场人、转型中的 PM）+ 激活（AI PM Bootcamp 已注册未付费学员） |
| **执行难度** | 🟢 小（无需开发，只需 Notion 公开页 + Google Form 收提交；讲师点评 3 篇约 2 小时） |
| **预估指标** | 10 天：200 人看到公告，60 人提交拆解，前 3 名上 LinkedIn 被动传播触达 2,000+；10 人预约 AI PM Bootcamp 试听，3 人付费（~$2k AUD/人） |
| **本周做？** | ⏳ 下周启动（Agent 365 GA 余热可持续 2–3 周；本周 ①②③ 已够 3 个高优先级，AI PM 拆解不如 B2B 和 Agent 拓扑赛紧迫；可本周准备，下周一发起） |

**落地步骤**：
1. **本周（D0–D2）**：JR AI PM 讲师写示范拆解（300 字，故意留 2 个明显的"改进空间"，引发讨论）；准备评审标准（用户同理心 / 定价逻辑 / 功能取舍清晰度）；设计投票机制（Notion 公开数据库，每条提交有 Like 按钮）
2. **D3（下周一发起）**：在 LinkedIn 发题目帖（JR AI PM 讲师署名）："Microsoft Agent 365 定价 $15/月，对还是错？500 字拆解，赢免费试听——" 附 Google Form 提交链接；截止 D8
3. **D4–D7**：每天在 LinkedIn 转发 1–2 条角度有趣的提交（获参与者同意，匿名发），带"你同意这个判断吗？"引发评论；评论多的帖子自然扩散
4. **D9**：社区投票 + 讲师评审，选前 3 名；发布「Top 3 Teardown 全文」合集帖（知乎 + LinkedIn），每份附讲师 100 字点评
5. **D10**：给所有提交者发「感谢参与」邮件 + AI PM Bootcamp 限时优惠（7 天有效期）；整理高质量拆解存入「AI PM 案例库」，用于课程教学素材（二次利用）

## 这周不推的玩法

- **「开源模型五选一 Hackathon」** — 上周竞品周报已推荐此类线下 Hackathon（Melbourne 3 小时），且上周增长玩法④已做模型盲测。本周三款新模型（Gemma 4 / Muse Spark / Mistral 128B）刚上线，API 稳定性未验证，现场跑 Demo 翻车风险高；内容 + 话题上和上周重叠，间隔太短效果打折。建议 4 周后做 Q2 综合测评版更成熟。
- **「学生半价认证计划」** — JR Bootcamp 客单价 $2k–$4k AUD，学生群体付费能力弱、转化链路长；且当前最热的 Microsoft/TAFE 培训受众是职场人而非在校生。做学生折扣只会占满销售精力，而客单价和完课率均低于职场人群体，ROI 不划算。
